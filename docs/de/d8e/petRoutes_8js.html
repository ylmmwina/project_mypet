Детальний опис

Маршрути API для управління улюбленцями.

  • Цей файл містить Express-обробники для створення, отримання списку, видалення улюбленців, а також виконання дій над ними (годування, гра, сон тощо).

◆ registerPetRoutes()

function default registerPetRoutes ( app,
db,
io )

Реєструє маршрути улюбленців у додатку Express.

Аргументи
{Object}app - Екземпляр додатка Express.
{Object}db - Екземпляр бази даних SQLite.
{Object}io - Екземпляр Socket.IO.

Оновлює стан улюбленця в базі даних.

  • Допоміжна функція, яка знаходить улюбленця, застосовує до нього дію (callback) і зберігає зміни.
  • Аргументи
    {string}ownerId - ID власника.
    {number}petId - ID улюбленця.
    {Function}actionCallback - Функція, що змінює параметри об'єкта Pet.
    Повертає
    {Promise<Object>} Оновлений об'єкт улюбленця (JSON).
    Обробка виняткових ситуацій
    {Error}Якщо улюбленця не знайдено або ID некоректний.
    Отримати всіх улюбленців користувача. @route GET /pets
    Аргументи
    {Object}req - Об'єкт запиту (req.ownerId встановлюється middleware).
    {Object}res - Об'єкт відповіді.
    Отримати улюбленця (альтернативний маршрут). @route GET /pet

Створити нового улюбленця. @route POST /create-pet

Аргументи
{Object}req - Тіло запиту містить { name, type }.
{Object}res - Повертає створеного улюбленця.

Завершення міні-гри. @route POST /pet/finish-game

  • Оновлює статистику (монети, щастя, голод) на основі результатів гри.
    Аргументи
    {Object}req - Тіло запиту { score, coinsEarned, petId }.
    Middleware для обробки стандартних дій (feed, play тощо).
    Аргументи
    {Function}actionCallback - Метод класу Pet.
    Погодувати улюбленця (POST /pet/feed)

Пограти з улюбленцем (POST /pet/play)

Вкласти спати (POST /pet/sleep)

Полікувати (POST /pet/heal)

Помити (POST /pet/clean)

Видалити улюбленця. @route POST /pet/delete

Аргументи
{Object}req - Тіло запиту { petId }.
